프로젝트 목표

항공기 내부를 실제 기체와 유사하게 제작하여, VR을 통해 항공기 계기판 및 조종 환경에 익숙해지는 것이 본 프로젝트의 목표입니다.

개발 일정 기획

사용자 요구사항 및 기능을 기반으로 제작된 개발 일정입니다.

프로젝트 구조

시스템 하드웨어 구조

사용자는 PC에서 입출력 도구 또는 VR장치를 통해 시뮬레이터에 접근할 수 있습니다.

시스템 소프트웨어 구조

프로젝트는 VR 또는 Non-VR환경에서 시뮬레이션 엔진에 접근하고, System UI/UX를 통해 사용자와 소통합니다.
사용자는 제공된 인터페이스를 통해 시뮬레이션 Main Level에 접근할 수 있고, Main Level에서는 가상 항공기에 접근할 수 있습니다.
가상 항공기는 사용자의 입력을 Virtual Joystick을 통해 신호를 받고, 해당 신호는 항공기의 Flight Computer에 전달됩니다.
Flight Computer는 항공기를 조종하는 함수에 접근할 수 있으며, 또한 항공기의 상태를 표시하는 항공 전자장비 객체에 접근 가능합니다.
항공기 객체는 항공기의 상태를 이용해 항공기의 애니메이션 및 소리, 충돌효과를 갱신합니다.
항공 전자장비 객체는 가진 상태를 System UI/UX에 전달하여 표현합니다.
VOR객체는 Main Level에서 수행됩니다. VOR은 항공기 객체에 접근해 위치 정보를 받아와 데이터를 가공하고, System UI/UX를 통해 사용자에게 전달됩니다.

Flowchart를 구성하여 실제 코드를 작성하기 이전에 로직을 구현했습니다.
